Stage 0 — Background Removal
BiRefNet v2 · fal.ai · ~3s

BiRefNet v2 (General Use Heavy)

Removes background from customer image. Uses bilateral reference framework for high-resolution segmentation. Runs when "BG Removal" toggle is ON.

API: fal-ai/birefnet/v2 Model: General Use (Heavy) Resolution: 1024×1024 Foreground refinement: ON Output: PNG with alpha channel
Stage 1 — Software Gates (10 deterministic checks)
$0 · <100ms · pixel math only

These gates run pixel-level analysis. They INFORM but don't decide — the VLM validates each flag.

BG-1 — No Alpha Channel

If BG removal was ordered but the image has no alpha channel → flag. Binary check.

flag = (bg_removal_on AND img.mode NOT in RGBA/LA/PA) confidence: 98% | authority: VLM-adjudicated BG removal OFF → skipped

BG-2 — White BG Corners

Samples 5% corner margins. If >92% of corner pixels are opaque white → flag as possible remaining background.

corners = 4 × (5% margin area) white = R>240, G>240, B>240, A>200 flag = (white_pct > 92%) FP rules: white design element, near-white on transparent

BG-3 — Content Border Opacity

Finds the content boundary (where design meets transparency via erosion). Checks what % of border pixels have alpha below 60% — these print as faint white halo.

content_border = content_mask AND NOT eroded_mask low_opacity = border_alpha < 153 (60%) flag = (low_opacity_pct > 15% AND count > 50) Physics: alpha < 60% = faint white ink deposit

BG-4 — Ghost Pixels

DPI-scaled dilation creates safe zone around main content. Orphan pixel clusters outside = ghost candidates.

dilated = cv2.dilate(content, kernel, iterations=3) ghost = component NOT overlapping dilated zone flag = (any cluster < 5px found outside safe zone) FP rule: intentional glow/sparkle effects

LR-1 — DPI / Resolution Check

DPI = pixels ÷ print inches. Below 72 = HARD FAIL (physics, no VLM override). 72-150 = borderline.

dpi = min(width_px / print_width_in, height_px / print_height_in) < 72 → HARD FAIL (visible pixelation guaranteed) 72-150 → borderline flag ≥ 150 → pass AUTHORITY: Hard block — no VLM override

LR-2 — Blur Detection

Laplacian variance on content region only (not mask boundary). Low variance = blurry image.

laplacian = cv2.Laplacian(gray, CV_64F) variance = laplacian[content_mask > 0].var() < 3 → definite blur | 3-10 → borderline | > 50 → sharp FP rules: shallow DOF, gradient/soft-brush art

TL-1 — Thin Line Detection

Finds ISOLATED narrow strokes (< 2px) separate from main design, surrounded by transparency. These can't hold adhesive powder.

connected_components = cv2.connectedComponentsWithStats() skip if component area > 10% of total (= main design) measure max_width via distance_transform flag if: max_width < 2px AND aspect_ratio > 3 (elongated) NOT: borders of filled shapes, connected text

CS-1 — CMYK Color Shift (Advisory)

Gamut-risk heuristic: identifies saturated colors (neon blue, vivid green, bright red) that will shift during CMYK conversion. Info-only, never flags.

risky_blue = hue 200-270° AND sat > 60% AND max > 100 risky_green = hue 80-170° AND sat > 60% risky_neon = sat > 85% AND max > 200 estimated ΔE based on saturation level AUTHORITY: Advisory only — never triggers FAIL Production: needs ICC profile for accuracy

JE-1 — Jagged Edge Detection

Angle-based analysis on contour points. Detects staircase patterns from bad upscaling.

contours = cv2.findContours(CHAIN_APPROX_NONE) angle_changes = arctan2(dy, dx) between consecutive points jagged = angle_change between 45°-135° (right-angle steps) flag = (jaggedness_ratio > 0.25 AND total_points > 100) FP rule: intentional pixel art / 8-bit style

QR-1 — QR Code Scan

Detects and decodes QR codes using OpenCV QRCodeDetector. Verifies readability.

detector = cv2.QRCodeDetector() decoded, points, straight = detectAndDecode(gray) Reports: QR found/not found, decoded data, is_url Multi-QR detection via detectAndDecodeMulti()
Stage 2 — VLM Assessment (4 checks + 7 validations)
~$0.01/img · 5-30s · fal.ai any-llm/vision

VLM receives TWO images (original + processed) and all SW gate results. It performs visual checks AND validates/overrides SW flags.

VLM-ONLY CHECKS (visual assessment)

UP-1 — Upscaling + Design Integrity

Compares original vs processed. Detects text corruption, face distortion, sharpening halos, objects incorrectly added/removed by AI upscaler.

Compares Image 1 (original) vs Image 2 (processed) FAIL: garbled text, melted faces, halo artifacts FAIL: objects added/removed not in original Customer: "number 1 in area code is messed up" Customer: "black ink smudges on the flag"

BR-1 — BG Removal Quality

6 specific defect sub-categories from BG removal research (Sprint 1/2).

a. ENCLOSED TEXT RESIDUAL — BG inside O,D,B,A,R,P,Q b. EDGE HALO / FRINGING — glow ring around subject c. HAIR / FINE-DETAIL LOSS — flyaway strands clipped d. INCOMPLETE REMOVAL — BG patches remaining e. COLOR BLEED — BG color into subject edges f. SHADOW ARTIFACTS — cast shadow retained Customer: "The L in Michael Jordan was removed" Customer: "white background behind the tree"

ST-1 — Semi-Transparency (DTF Physics)

DTF CANNOT reproduce gradual transparency. Any glow/smoke/shadow with partial opacity = white/gray haze on garment. Intent does NOT matter.

FAIL: glow/smoke effects with gradual fade FAIL: neon outlines fading to nothing FAIL: drop shadows with transparency FAIL: faded/feathered edges to transparent FAIL: cloud/fog semi-transparent areas PASS: only if ALL elements fully opaque Customer: "Shows white gray and brown after pressed" #1 source of "doesn't look like what I ordered"

TL-VLM — Thin Lines (Visual)

Isolated thin elements SEPARATE from main design, SURROUNDED by transparency after BG removal, < 2px. Can't hold adhesive powder.

IS thin line: separate stroke floating in transparent space IS thin line: outline separated from fill after BG removal NOT thin line: edges of filled shapes in main design NOT thin line: connected text (even if strokes thin) Customer: "yellow line didn't press to the shirt" Customer: "no powder on the film in some areas"

Trimming / Sizing

Spelling, placement, formatting, color matches customer request. Verify across all images in order.

Color Matching

Exact color codes if customer specified them (e.g., "navy blue #063970"). Colors match across all redraws.
